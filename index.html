📖 Workflow — Intégrer un nouveau catalogue labo

1. Préparer le document tarifaire — PDF avec : références (EAN, désignation, prix HT, TVA si dispo), conditions de remise (marchés, paliers, UG), franco / minimum de commande. Sources : CGV annuelles, tarif général, bon de commande groupement, grille délégué.

2. Demander l'intégration à Claude — uploader le PDF avec ce message :

« Intègre le catalogue [NOM DU LABO] avec ce PDF, sur le même modèle que TOPICREM/BAYER/THEA : produits pré-chargés + marchés de remises. Récupère d'abord la version stable depuis GitHub et vérifie que les fixes précédents sont présents. Teste avec Node avant de me livrer. »

Vérifier dans la réponse : nombre de références, marchés et paliers, anomalies signalées (TVA estimées, paliers ambigus, offres datées) → points à confirmer avec le délégué.

3. Télécharger le fichier — l'index.html livré, sur le Mac de déploiement. Vérifier le nom : ls -lt ~/Downloads/index*.html (prendre le plus récent ; si « index (1).html », adapter le cp).

4. Déployer :

cd ~/calculateur-pharmacie
git pull origin main
cp ~/Downloads/index.html index.html
git add index.html
git commit -m "feat: catalogue [LABO] XX refs"
git push origin main

Si pull refusé (divergence) : git fetch origin && git reset --hard origin/main puis cp / add / commit / push. Machine sans repo : git clone https://github.com/pharmaciemichelet127-cyber/calculateur-pharmacie.git d'abord. Attendre 1-2 min (Vercel).

5. Activer dans l'app — ① Cmd+Shift+R (obligatoire, sinon ancien code en cache) · ② si machine vierge : « Charger depuis GitHub » d'abord · ③ « + Nouveau labo », nom exact et sans accent (THEA, pas THÉA) · ④ cliquer la pastille du labo — c'est ce clic qui charge le catalogue · ⑤ onglet Catalogue : vérifier les références · ⑥ « Sauvegarder sur GitHub » → ✓.

6. Contrôles avant le RDV — pointer 2-3 prix contre le PDF · vérifier les TVA si estimées · confirmer les points flaggés avec le délégué · saisir les conditions négociées dans ⚡ Décision RDV.

Lexique :

↺ Pré-remplir (offre UPP)Recopie dans les Conditions tout ce que contient l'offre du labo (marchés, mots-clés, taux, franco), avec les taux du palier objectif. C'est un « retour à l'offre » : si les cases contiennent déjà l'offre, cliquer ne change rien ; si tu as modifié des valeurs à la main, cliquer les efface pour revenir à l'offre officielle (confirmation demandée). Jamais de cumul : 10 clics = 1 clic.
✓ ACQUISLe plus haut palier d'engagement que ton rythme de vente OTC couvre déjà, sans aucun effort. C'est ta position de repli en négociation.
🎯 OBJECTIFLe palier supérieur quand il est à moins de 20% d'effort de ton rythme. Le plan de commande et le pré-remplissage des conditions visent ce palier. L'écart entre ACQUIS et OBJECTIF = ce que le délégué doit t'aider à financer.
Périmètre offreLes seuls produits cités dans l'offre du labo. Le verdict, le volume d'engagement et les remises se calculent dessus — jamais sur tout ton CA labo (les remboursés hors offre à 0% dilueraient tout).
Bonus remboursésRemises croisées sur les remboursés cités (ex. Théa : Monoprost, Théalose…). De l'argent réel, mais jamais le critère de choix du marché — le choix se fait sur l'absorption du volume OTC.
Badge versionLa pastille en bas à droite. Si elle diffère du numéro annoncé par Claude : Cmd+Shift+R, ou attendre 1-2 min que Vercel déploie.
Charger / SauvegarderGitHub est la source de vérité des données. Sur chaque Mac : Charger avant de travailler, Sauvegarder après. Jamais Sauvegarder depuis une machine qui affiche moins de labos que la réalité.

Marge compensee AVENE — Workflow & Strategie

De l export Ospharm a la decision de prix

Objectif final

Tu vends 299 references AVENE. La marge compensee consiste a accepter une marge reduite sur les produits tres exposes (TS), tenir une marge correcte sur les sensibles (S), et recuperer la marge perdue sur les produits peu compares (PS). Resultat : un MB% global cible de 35% avec des prix competitifs la ou ca compte.

Etape 1 — Export Ospharm "Prix de proximite"

Ce qu on fait : exporter depuis Ospharm le fichier Top de proximite AVENE (robot Python ou export manuel).
Pourquoi : connaitre les prix pratiques par les pharmacies de quartier comme toi a Marseille — colonne Quartier.
Ce qu on obtient : PMC Prox, Quartier, Max Zone, Panel National par EAN.

Colonne cle : Quartier = reference pour fixer tes prix TS.

Etape 2 — Export Quartz (.xlsx)

Ce qu on fait : exporter la liste AVENE avec sensibilite depuis QuartzPharma.
Pourquoi : savoir quels produits tes clients comparent activement.

Etape 3 — Etat des ventes LGPI (.pdf)

Ce qu on fait : exporter l etat des ventes AVENE 12 mois en PDF depuis LGPI.
Pourquoi : obtenir PA HT reel, PV actuel et rotation mensuelle (QTE 12 mois / 12).
Ce qu on obtient : donnees fusionne sur EAN avec Quartz.

Rotation = 0 apres import = produit non vendu ou EAN non matche. Impact nul sur la marge estimee.
